Fundamentos e Introdução a Frameworks

A base do curso foi estabelecida com a revisão das motivações e fundamentações do desenvolvimento mobile, o entendimento das diferenças entre as abordagens nativa e híbrida, e a confirmação das ferramentas essenciais: HTML, CSS e JavaScript.

O próximo passo é aprofundar no desenvolvimento, com foco em plataformas e frameworks. As questões centrais a serem exploradas são: O que é um framework?, O que é híbrido? e, por fim, qual a melhor abordagem em um cenário com diversas opções.

É fundamental que se entenda a filosofia de que nunca existiu um framework único ou universalmente superior. O mundo do desenvolvimento é marcado pela mistura de tecnologias e múltiplas abordagens, e o aluno deve estar preparado para essa diversidade.

Seguindo esta base, o curso focará em um framework baseado em Node.js. Assim, as próximas etapas serão dedicadas a compreender, instalar e utilizar o Svelte.

Configuração e Inicialização do Projeto Svelte

Esta etapa detalha a instalação do ambiente de desenvolvimento (Node.js) e a criação de um novo projeto usando ferramentas de linha de comando (CLI), focado no Svelte.

  1. Instalação do Node.js
    • O desenvolvimento híbrido baseado em JavaScript exige a instalação do Node.js.
      • O Node.js permite a criação de uma estrutura embarcada (aplicativos mobile) baseada na engine V8 do navegador Chrome.
      • A instalação é simples: basta baixar e executar o instalador do Node.js.
      • Ao instalar o Node.js, o NPM (Node Package Manager), o gerenciador de pacotes, é instalado automaticamente.
  2. Instalação do Svelte CLI (Adaptação)
    • Embora o material mencione a instalação do Ionic CLI, para utilizar o Svelte, tipicamente utilizamos ferramentas como npm ou yarn para inicializar o projeto diretamente, ou um pacote CLI relacionado ao ecossistema Svelte (como o vite para criar projetos Svelte).
    • Assumindo a adaptação: A instalação do CLI (Command Line Interface) do framework seria feita via npm, utilizando a opção -g para instalação global.
    • Nota de Permissão: Pacotes globais podem causar erros de permissão (EACCES). É recomendado configurar o npm para operar globalmente sem a necessidade de permissões elevadas.
  3. Criando e Executando um Aplicativo Inicial
    • Com o ambiente configurado, o próximo passo é gerar o aplicativo Svelte inicial.
      1. Criação do Aplicativo: O comando npm create vite@latest (ou um comando similar para inicialização de Svelte) inicia um processo interativo que solicita detalhes de configuração do APP.
      2. Inicialização: Após a instalação dos pacotes, entre no diretório do novo projeto.
        • cd [nome-do-app]
      3. Execução do Servidor: Use o comando de serve do framework (ex.: npm run dev) para iniciar o aplicativo.
        • Como se trata de uma versão híbrida, o aplicativo será executado no navegador em um endereço local (http://localhost:[porta]).

Ao executar, o APP inicial é simples (normalmente com navegação básica entre telas), mas demonstra o grande avanço da automação no processo de desenvolvimento híbrido.

Entendendo a Interatividade e Primeiras Alterações

Esta seção aborda a verificação e a alteração do código-fonte do aplicativo em tempo real, garantindo a compreensão do ciclo de desenvolvimento.

Interatividade em Tempo Real

  1. Abertura do Projeto: Para começar a editar o código-fonte, o ambiente de desenvolvimento (VS Code, mencionado anteriormente) deve ser aberto na pasta do projeto usando o comando de terminal: code .
  2. Edição Rápida (Hot Reload): O principal recurso a ser notado é que qualquer alteração feita no código-fonte é refletida imediatamente na saída de dados no navegador (função conhecida como Hot Reload ou Live Reload).
  3. Detecção de Erros: Caso uma alteração introduza um erro no código, a tela de visualização no navegador reagirá, geralmente ficando em branco ou cinza, sinalizando o problema.

Trilha Prática: Alterando o Exemplo Base

A prática consiste em customizar o aplicativo inicial, alterando os nomes das abas (ou tabs) de navegação.

Objetivo: Modificar os nomes padrão para algo mais significativo: Aluno, Secretaria e Calendário.

  1. Alteração de Conteúdo: O texto sugere modificar a propriedade name de um componente de exibição em arquivos de página (como tab1-page.html, tab2-page.html, etc. — que, no contexto Svelte, seriam provavelmente componentes .svelte).
    • Exemplo de alteração conceitual:
      • Página 1: name = "Aluno"
      • Página 2: name = "Secretaria"
      • Página 3: name = "Calendário"
  2. Alteração da Barra de Navegação: É necessário também alterar os rótulos (labels) que aparecem na barra de navegação inferior (que, no Svelte, seriam componentes de interface). A ideia é mapear os rótulos Aluno, Secretaria e Calendário aos seus respectivos ícones e tabs.

Após as alterações, o aluno testa o aplicativo no navegador para ver o resultado da customização.

Conclusão da Etapa

O objetivo principal deste conteúdo é praticar as primeiras configurações e alterações em um projeto mobile híbrido, exercitando a familiaridade com o ambiente de desenvolvimento e o ciclo de feedback rápido.
