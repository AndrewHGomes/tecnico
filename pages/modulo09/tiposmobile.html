Tipos de Desenvolvimento Mobile

Introdução e Tipos de Desenvolvimento Mobile

O desenvolvimento mobile teve um crescimento exponencial, impulsionado pela Indústria 4.0 e pela necessidade de diversas formas de acesso. Atualmente, a fronteira entre o desenvolvimento mobile e web é fluida, com muitas ferramentas servindo para ambos os ambientes. Existem três abordagens principais para o desenvolvimento de aplicativos móveis:

  1. Desenvolvimento Nativo: Criação de um aplicativo específico para a plataforma (ex: Android usando Kotlin/Java, iOS usando Swift/Objective-C).
  2. Desenvolvimento Híbrido: Uso de tecnologias primariamente web (como HTML, CSS, JavaScript) que são empacotadas para distribuição em lojas (App Store, Google Play).
  3. Desenvolvimento Web (Web Mobile): Criação de um site otimizado para navegadores móveis (rodando em um browser no celular).

Diferenças e Comparação entre Abordagens

  1. Desenvolvimento Nativo
    • Vantagem: Permite a execução e o desempenho máximo no ambiente nativo.
    • Desvantagem: Exige um novo desenvolvimento para cada plataforma (Android, iOS).
  2. Desenvolvimento Híbrido
    • Vantagens:
      • Custo Baixo: Não exige linguagens próprias de cada plataforma.
      • Migração Facilitada: Versão única para diferentes sistemas operacionais.
      • Manutenção Simplificada.
      • Pode usar plugins para acessar recursos do celular (câmera, etc.).
    • Desvantagens:
      • Performance Inferior: Possui mais camadas internas para se comunicar com a plataforma nativa.
      • Limitações de Design e Funcionalidades: Menos recursos de interface que o nativo; muitas vezes exige estar em primeiro plano e on-line.
      • Usabilidade: A experiência pode ser comprometida por ser a mesma versão para sistemas com usabilidades distintas (iOS vs. Android).
  3. Desenvolvimento Web Mobile
    • Vantagem: Roda em qualquer browser no celular.
    • Desvantagem: Não pode ser empacotado para distribuição em lojas e tem limitações no acesso direto aos recursos do sistema (hardware) do celular.

Foco para a Escolha

A escolha da abordagem deve considerar o investimento disponível, o tempo de desenvolvimento planejado e, principalmente, as funcionalidades e a necessidade de desempenho. O sucesso do aplicativo depende de garantir uma excelente experiência do usuário, focada em agilidade de carregamento e interface simples e interativa.

Infraestrutura e Ferramentas para Desenvolvimento

Para iniciar o desenvolvimento mobile, é necessário configurar a infraestrutura de ferramentas:

  • Desenvolvimento Nativo Android: É obrigatório o uso do Android Studio, uma plataforma de desenvolvimento que exige, no mínimo, 8 GB de memória RAM para funcionar de forma eficiente devido ao seu peso.
  • Desenvolvimento Híbrido: Requer um editor de código-fonte livre, como o VSCode (Visual Studio Code).

Em ambos os casos, o Kit de Desenvolvimento Android (SDK) deve ser instalado para que seja possível empacotar o aplicativo e ter acesso a todas as funcionalidades do sistema operacional.

HTML e CSS (Conceitos Básicos)

O desenvolvimento mobile, especialmente o híbrido, aproveita tecnologias web como HTML, CSS e JavaScript, que são então encapsuladas.

HTML (HyperText Markup Language)

O HTML é a linguagem de marcação usada para estruturar o conteúdo de uma página (ou interface de aplicativo, no caso híbrido).

  • Ferramenta: Para começar, basta um editor de texto simples (sem formatação, diferente do Word).
  • Processo Básico: Você digita o código HTML e salva o arquivo com a extensão .html. Ao dar um duplo clique, ele será aberto e interpretado por um navegador, que está amarrado à extensão.
  • Referência: O W3CSchool é uma fonte fundamental para aprender e experimentar códigos HTML.
  • Conceito: O HTML funciona como um arquivo-texto com marcações (tags) que definem a estrutura, como parágrafos, títulos e links.

CSS (Cascading Style Sheets)

O CSS é a linguagem usada para formatar e estilizar o conteúdo estruturado pelo HTML.

  • Função: Ele é responsável por toda a parte visual, como cores, fontes, layouts e posicionamento dos elementos na tela.
  • Importância: É essencial para garantir a aparência e a experiência visual do aplicativo ou página, sendo usado em conjunto com o HTML.

JavaScript e TypeScript

JavaScript

O JavaScript é uma das linguagens de programação mais populares, sendo a base do desenvolvimento Front-End na web, responsável por adicionar dinamismo às páginas.

  • Função: Permite incluir elementos dinâmicos (mapas, formulários, animações, cálculos numéricos) e dar ações ao sistema, como conectar o aplicativo e implementar funcionalidades.
  • Expansão: Com a criação do Node.js, o JavaScript se expandiu para o desenvolvimento Back-End, permitindo que muitos programadores trabalhem em um projeto inteiro utilizando apenas uma linguagem.
TypeScript

Embora não detalhado, o conceito de TypeScript está ligado a essa evolução, sendo um superset (extensão) do JavaScript que adiciona tipagem estática.

Vantagens dos Aplicativos Híbridos (Reforço)

Aprender e utilizar HTML, CSS e JavaScript é fundamental para o desenvolvimento híbrido devido às suas vantagens principais:

  1. Baixo Custo de Desenvolvimento: Menor investimento de tempo e financeiro.
  2. Menor Curva de Aprendizado: Exige menos conhecimento técnico especializado em linguagens nativas.
  3. Flexibilidade (Cross-Platform): O mesmo código-fonte atende às principais plataformas do mercado (iOS e Android).
  4. Facilidade de Atualizações e Manutenção.
  5. Ideal para Protótipos: Permite a construção rápida de aplicativos-teste.

O uso dessas linguologias web, junto a Frameworks modernos, oferece um fluxo de desenvolvimento muito rápido. O foco agora é entender os conceitos para aplicá-los futuramente.
